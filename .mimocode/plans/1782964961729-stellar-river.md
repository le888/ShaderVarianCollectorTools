# 重构计划：ShaderVariantCollectorWindow 从 UXML/UIElements 迁移到 IMGUI

## 目标

将 `ShaderVariantCollectorWindow` 的 UI 从 UXML + UIElements 改为纯 IMGUI（`OnGUI` + `EditorGUILayout`），删除不再需要的 UXML 文件和 `PathSelector` 组件。

## 需要修改的文件

| 文件 | 操作 |
|---|---|
| `Editor/ShaderVariantCollector/ShaderVariantCollectorWindow.cs` | **重写** UI 层为 IMGUI |
| `Editor/ShaderVariantCollector/PathSelector.cs` | **删除**（VisualElement 子类，IMGUI 不需要） |
| `Editor/ShaderVariantCollector/PathSelector.cs.meta` | **删除** |
| `Editor/ShaderVariantCollector/ShaderVariantCollectorWindow.uxml` | **删除** |
| `Editor/ShaderVariantCollector/ShaderVariantCollectorWindow.uxml.meta` | **删除** |
| `Editor/UxmlLoader.cs` | **不动**（其他窗口可能还在用） |

### IMGUI 辅助方法
```csharp
// 替代 PathSelector 的文件夹选择
private string FolderPathField(string label, string currentPath)
{
    EditorGUILayout.BeginHorizontal();
    string newPath = EditorGUILayout.TextField(label, currentPath);
    if (GUILayout.Button("选择", GUILayout.Width(60)))
    {
        string selected = EditorUtility.OpenFolderPanel("选择文件夹", "Assets", "");
        if (!string.IsNullOrEmpty(selected) && selected.Contains("/Assets"))
            newPath = EditorTools.AbsolutePathToAssetPath(selected);
    }
    EditorGUILayout.EndHorizontal();
    return newPath;
}
```

### 动态列表渲染模式
每个列表（黑名单、过滤着色器、关键字）用 `Foldout` 折叠，内部循环绘制每项，底部提供输入框和添加按钮。删除按钮用 `GUILayout.Button("X")` 实现。

1. 所有路径选择功能正常（点击选择按钮能弹出文件夹选择器）

3. 设置修改后能正确持久化（关闭窗口重新打开，值还在）
4. 收集按钮能正常触发收集流程
